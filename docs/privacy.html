UptimeBot Privacy Policy

Effective date: February 28, 2026

Last updated: February 27, 2026

Terms of Service: View Terms

UptimeBot is operated by an independent individual developer and provides Discord bot uptime and downtime tracking features.

Before using UptimeBot, please review these Privacy terms together with the Terms of Service.

1. Who Operates UptimeBot

Operator: UptimeBot (independent individual developer)

Contact Email: contact@uptimebotdiscord.com

Support: Support Server

UptimeBot is not affiliated with, endorsed by, or sponsored by Discord Inc.

2. Summary

In short:

  • UptimeBot processes limited Discord metadata needed for core features
  • UptimeBot does not sell personal data
  • Deletion can be requested at any time using the contact details below

3. Scope

This Privacy Policy applies to data processed by UptimeBot in Discord servers where the bot is installed.

4. Data Processed by UptimeBot

A. Data stored in the service database

  • ServerID (Discord server ID)
  • pingChannel (configured alert channel ID)
  • premium flag (server premium status)
  • Watchlist records: UserID (watching user's Discord ID), ServerID, and Bots (tracked bot IDs)

B. Data processed in memory (not intended for long-term storage)

  • Presence and status events used to detect online/offline changes
  • Debounce and session state for alerts and menu interactions

C. Operational logs

  • Error and diagnostic logs may include Discord IDs and technical context

5. Purpose of Processing

  • Deliver uptime and downtime alerts
  • Manage watchlists and server settings
  • Prevent duplicate or spammy alerts
  • Maintain, debug, and secure the service

6. Use of UptimeBot in Discord

If multiple users in a server track the same bot, UptimeBot may send one alert embed with multiple user mentions in a single message (or batches if needed for Discord limits).

UptimeBot is provided for general informational purposes and is not a mission-critical monitoring system. Alerts may be delayed, incomplete, or affected by Discord API limitations.

UptimeBot does not store or archive message content from Discord servers.

7. Sharing of Data

UptimeBot does not sell personal data.

Data may be processed by:

  • Discord (platform and API)
  • Hosting and database providers used to run UptimeBot
  • Legal authorities when required by law

8. Data Retention and Deletion

  • Server watchlist and settings data is automatically deleted 7 days after UptimeBot is removed from that server
  • Data can be deleted earlier by request
  • Operational logs are retained only as needed for reliability and security

To request deletion, email contact@uptimebotdiscord.com and include Data Removal Request in the subject line.

9. Your Choices

  • Remove tracked bots from your watchlist
  • Remove UptimeBot from your server
  • Request deletion by email with subject line Data Removal Request

10. Security

Reasonable technical and organizational safeguards are used, but no system can be guaranteed 100% secure.

11. Children's Privacy

Users must meet the minimum age required by Discord in their jurisdiction.

12. International Processing

Data may be processed in countries other than your own, depending on infrastructure providers.

13. Changes to This Policy

This policy may be updated from time to time. Continued use after updates means acceptance of the revised policy.

14. Contact

For questions, contact contact@uptimebotdiscord.com . For deletion requests, include Data Removal Request in the subject line.

For general community support, use the UptimeBot Support Server
